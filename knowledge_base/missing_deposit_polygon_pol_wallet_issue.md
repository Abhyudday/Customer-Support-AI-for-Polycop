# Missing Deposit Issue - POL Tokens Sent to Wrong Address

## Problem
User sent POL tokens from Phantom Wallet to Polycop Bot deposit wallet.

## Symptoms
- Telegram bot shows balance as $0
- PolygonScan shows correct funds
- Funds are not reflected inside the bot

## Cause
POL tokens were sent to the wrong address.

The Polycop deposit wallet does NOT support direct POL token deposits.
Users must send funds to the bridge deposit address instead.

## Solution
The funds are not lost, but require manual recovery.

Steps:
1. Visit the official Polymarket recovery guide:
   https://docs.polymarket.com/polymarket-learn/FAQ/recover-missing-deposit

2. Follow the instructions to:
   - Convert the tokens correctly
   - Withdraw them back to a usable wallet

3. After recovery:
   - Use the correct bridge deposit address for future deposits

## Important Notes
- The deposit wallet is a sub-wallet generated by Polymarket
- Direct access to this wallet is not available
- Incorrect chain/token deposits will not be auto-detected by the bot

## Recommended Practice
Always send funds using the correct bridge deposit method to avoid this issue.
